# Техники тест-дизайна

## Применение техник тест-дизайна в проекте "BSPU PK Base"

### Используемые техники:
- **EP** - Эквивалентное Разделение (Equivalence Partitioning)
- **BVA** - Анализ Граничных Значений (Boundary Value Analysis)
- **CE** - Причина / Следствие (Cause/Effect)
- **EG** - Предугадывание ошибки (Error Guessing)
- **DT** - Таблица принятия решений (Decision Table)

---

## 1. Поиск УОСО

### Требования:
- Поле поиска принимает текстовый ввод
- Поиск выполняется в реальном времени
- Максимальная длина не указана (предполагаем разумные ограничения)

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Корректные названия школ (например: "школа", "гимназия", "лицей")
- **Класс 2 (OK)**: Частичные совпадения (например: "шк", "гим")
- **Класс 3 (NOK)**: Несуществующие названия (например: "xyz123", "абвгд")

#### BVA (Анализ Граничных Значений):
- **OK**: 1 символ (минимальная длина для поиска)
- **OK**: 50 символов (типичная длина названия)
- **OK**: 200 символов (длинное название)
- **NOK**: Пустая строка (если поиск не должен работать с пустым значением)
- **NOK**: 10000+ символов (очень длинная строка - EG)

#### EG (Предугадывание ошибки):
- Специальные символы: `!@#$%^&*()`
- SQL-инъекции: `' OR '1'='1`
- XSS: `<script>alert('test')</script>`
- Unicode символы: `школа`, `école`
- Пробелы в начале/конце: `  школа  `
- Только пробелы: `     `

### Тестовые данные:

| Значение | Ожидаемый результат | Техника |
|----------|---------------------|---------|
| "школа" | OK - найдены школы с "школа" в названии | EP |
| "гимназия" | OK - найдены гимназии | EP |
| "xyz123" | NOK - "Результаты не найдены" | EP |
| "" (пусто) | OK/NOK - зависит от требований | BVA |
| "ш" | OK - найдены школы с "ш" в начале | BVA |
| "!@#$%^&*()" | NOK - "Результаты не найдены"  | EG |
| `<script>alert('test')</script>` | NOK - "Результаты не найдены" | EG |
| "  школа  " | OK - поиск "школа" | EG |

## 2. Авторизация в админке

### Требования:
- Email: обязательное поле, формат email
- Пароль: обязательное поле, минимальная длина не указана

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Валидный email + валидный пароль
- **Класс 2 (NOK)**: Невалидный email + валидный пароль
- **Класс 3 (NOK)**: Валидный email + невалидный пароль
- **Класс 4 (NOK)**: Пустые поля

#### BVA (Анализ Граничных Значений) для Email:
- **OK**: `user@example.com` (стандартный формат)
- **OK**: `user.name@example.com` (с точкой)
- **OK**: `user+tag@example.co.by` (с плюсом и поддоменом)
- **NOK**: `user@` (без домена)
- **NOK**: `@example.com` (без имени пользователя)
- **NOK**: `user@example`

#### BVA (Анализ Граничных Значений) для Пароля:
- **OK**: 8 символов (типичный минимум)
- **OK**: 16 символов (стандартная длина)
- **OK**: 64 символа (максимальная разумная длина)
- **NOK**: Пустой пароль
- **NOK**: 1 символ (слишком короткий - EG)

#### EG (Предугадывание ошибки):
- SQL-инъекция в email: `admin'--@example.com`
- XSS в email: `<script>@example.com`
- Очень длинный email: `a@b.c` + 1000 символов
- Специальные символы в пароле: `P@ssw0rd!`
- Пробелы в email: `user name@example.com`
- Кириллица в email: `пользователь@example.com`

### Тестовые данные:

| Email | Пароль | Ожидаемый результат | Техника |
|-------|--------|---------------------|---------|
| `admin@example.com` | `password123` | OK - успешная авторизация | EP |
| `wrong@example.com` | `password123` | NOK - ошибка авторизации | EP |
| `admin@example.com` | `wrongpass` | NOK - ошибка авторизации | EP |
| `` (пусто) | `` (пусто) | NOK - валидация полей | BVA |
| `user@` | `password123` | NOK - валидация email | BVA |
| `@example.com` | `password123` | NOK - валидация email | BVA |
| `admin@example.com` | `` (пусто) | NOK - валидация пароля | BVA |
| `admin'--@example.com` | `password123` | NOK - SQL-инъекция блокирована | EG |
| `<script>@example.com` | `password123` | NOK - XSS блокирован | EG |

## 3. Добавление школы через админку

### Требования (предполагаемые):
- Название школы: обязательное, 1-200 символов
- Регион: обязательное, выбор из списка
- Район: обязательное, выбор из списка (зависит от региона)
- Тип УОСО: обязательное, выбор из списка

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Все обязательные поля заполнены корректно
- **Класс 2 (NOK)**: Одно из обязательных полей пустое
- **Класс 3 (NOK)**: Невалидные значения в полях

#### BVA (Анализ Граничных Значений) для Названия:
- **OK**: 1 символ (минимальная длина)
- **OK**: 100 символов (типичная длина)
- **OK**: 200 символов (максимальная длина)
- **NOK**: Пустое поле
- **NOK**: 201 символ (превышение максимума)

#### CE (Причина/Следствие):
- **Причина**: Заполнение всех обязательных полей + нажатие "Сохранить"
- **Следствие**: Школа создана, отображается в списке, ID присвоен

#### EG (Предугадывание ошибки):
- SQL-инъекция в названии: `Школа'; DROP TABLE schools;--`
- XSS в названии: `<script>alert('XSS')</script>`
- Очень длинная строка: 10000+ символов
- Специальные символы: `Школа №1 "Победа" (г.Минск)`
- Только пробелы: `     `
- Unicode символы: `Школа 学校 École`

### Тестовые данные:

| Название | Регион | Район | Тип УОСО | Ожидаемый результат | Техника |
|---------|--------|-------|----------|---------------------|---------|
| "Средняя школа №1" | Минская | Минский | Школа | OK - школа создана | EP |
| "" (пусто) | Минская | Минский | Школа | NOK - валидация | BVA |
| "Средняя школа №1" | "" (пусто) | Минский | Школа | NOK - валидация | BVA |
| "А" (1 символ) | Минская | Минский | Школа | OK - школа создана | BVA |
| "А"*200 (200 символов) | Минская | Минский | Школа | OK - школа создана | BVA |
| "А"*201 (201 символ) | Минская | Минский | Школа | NOK - превышение лимита | BVA |
| `Школа'; DROP TABLE;--` | Минская | Минский | Школа | NOK - SQL-инъекция блокирована | EG |
| `<script>alert('XSS')</script>` | Минская | Минский | Школа | NOK - XSS блокирован | EG |

## 4. Фильтрация по региону/району

### Требования:
- Фильтр "Регион": выбор из списка
- Фильтр "Район": активируется после выбора региона
- Фильтр "УОСО": активируется после выбора района

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Выбор первого региона в списке
- **Класс 2 (OK)**: Выбор последнего региона в списке
- **Класс 3 (OK)**: Выбор региона из середины списка
- **Класс 4 (OK)**: Сброс фильтров (кнопка "Все области")

#### BVA (Анализ Граничных Значений):
- **OK**: Первый регион в списке (граница начала)
- **OK**: Последний регион в списке (граница конца)
- **OK**: Регион с минимальным количеством школ (1 школа)
- **OK**: Регион с максимальным количеством школ (много школ)

### Тестовые данные:

| Действие | Ожидаемый результат | Техника |
|---------|---------------------|---------|
| Выбрать первый регион | OK - фильтр района активирован | BVA |
| Выбрать последний регион | OK - фильтр района активирован | BVA |
| Быстро переключать регионы | OK - нет ошибок, корректная фильтрация | EG |

## 5. Управление баннерами - период активности

### Требования:
- Дата начала: обязательное поле
- Дата окончания: обязательное поле
- Дата окончания должна быть >= даты начала

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Дата начала < даты окончания
- **Класс 2 (OK)**: Дата начала = дате окончания
- **Класс 3 (NOK)**: Дата начала > даты окончания

#### BVA (Анализ Граничных Значений):
- **OK**: Начало = сегодня, окончание = завтра
- **OK**: Начало = сегодня, окончание = сегодня (равны)
- **OK**: Начало = вчера, окончание = сегодня (прошедший период)
- **OK**: Начало = завтра, окончание = послезавтра (будущий период)
- **NOK**: Начало = завтра, окончание = сегодня (неверный порядок)
- **NOK**: Начало = сегодня, окончание = вчера (неверный порядок)

#### EG (Предугадывание ошибки):
- Очень старые даты: `1900-01-01`
- Очень будущие даты: `2100-12-31`
- Несуществующие даты: `2024-02-30`
- Високосный год: `2024-02-29` (OK), `2023-02-29` (NOK)
- Разные форматы: `31.12.2024`, `2024/12/31`
- Время вместо даты: `12:00:00`

### Тестовые данные:

| Дата начала | Дата окончания | Ожидаемый результат | Техника |
|------------|----------------|---------------------|---------|
| `2024-01-01` | `2024-12-31` | OK - баннер создан | EP |
| `2024-01-01` | `2024-01-01` | OK - баннер на 1 день | BVA |
| `2024-12-31` | `2024-01-01` | NOK - валидация дат | EP |
| `2024-01-01` | `2023-12-31` | NOK - валидация дат | BVA |
| `2024-02-29` | `2024-03-01` | OK - високосный год | EG |
| `2023-02-29` | `2023-03-01` | NOK - не високосный год | EG |
| `1900-01-01` | `1900-12-31` | OK/NOK - зависит от требований | EG |

## 6. Статистика по годам

### Требования (предполагаемые):
- Учебный год: обязательное, уникальное значение
- Формат: `2024/2025`
- Данные по классам: числовые значения

### Применение техник:

#### EP (Эквивалентное Разделение):
- **Класс 1 (OK)**: Уникальный учебный год
- **Класс 2 (NOK)**: Дублирующий учебный год
- **Класс 3 (OK)**: Валидный формат года

#### BVA (Анализ Граничных Значений) для года:
- **OK**: `2020/2021` (прошлый год)
- **OK**: `2024/2025` (текущий год)
- **OK**: `2025/2026` (будущий год)
- **NOK**: Неверный формат `2024-2025`
- **NOK**: Неполный формат `2024/`


#### BVA (Анализ Граничных Значений) для количества классов:
- **OK**: 0 (нет классов)
- **OK**: 1 (минимум)
- **OK**: 10 (типичное значение)
- **OK**: 100 (максимум)
- **NOK**: Отрицательное значение `-1`
- **NOK**: Дробное значение `10.5`
- **NOK**: Текст вместо числа `abc`

#### EG (Предугадывание ошибки):
- Очень большие числа: `999999999`
- Специальные символы: `2024/!@#`
- Пустые значения в числовых полях

### Тестовые данные:

| Учебный год | Количество классов | Ожидаемый результат | Техника |
|-------------|-------------------|---------------------|---------|
| `2024/2025` | 10 | OK - запись создана | EP |
| `2024/2025` | 9 | NOK - дублирование года | EP |
| `2024-2025` | 10 | NOK - неверный формат | BVA |
| `2024/2025` | 0 | OK - запись создана | BVA |
| `2024/2025` | -1 | NOK - валидация отрицательного | BVA |
| `2024/2025` | `abc` | NOK - валидация типа данных | EG |
| `2024/!@#` | 10 | NOK - Спец символы | EG |
